# Codebase Structure

**Analysis Date:** 2026-04-14

## Directory Layout

```
dsde_final_project/
├── election_pipeline/     # Primary production Airflow system
│   ├── dags/             # Airflow DAG definitions
│   ├── src/              # Core processing logic and utilities
│   ├── credentials/       # Storage for Google Drive API tokens
│   └── output_data/      # Structured output storage (CSV/JSON)
├── thai-trocr/            # Experimental Transformer-based OCR tests
├── typhoon/               # Experimental Typhoon VLM notebooks
├── wikipedia-scraped/     # Scripts for scraping candidate master data
├── .planning/             # GSD framework project state and docs
├── pyproject.toml         # Root project configuration (uv)
└── README.md              # Project overview and usage guide
```

## Directory Purposes

**election_pipeline/dags/**
- Purpose: Airflow orchestration entry points.
- Contains: `*.py` DAG files.
- Key files: `election_dag.py` - The main OCR pipeline DAG.

**election_pipeline/src/**
- Purpose: Modular business logic and processing utilities.
- Contains: `*.py` module files.
- Key files:
  - `processor.py` - Image processing and API integration.
  - `ocr_parser.py` - Markdown parsing and data validation.
  - `config.py` - Configuration and master lists.
  - `gdrive_client.py` - GDrive API wrapper.
  - `exporter.py` - Data persistence logic.

**thai-trocr / typhoon /**
- Purpose: Research and development workspaces.
- Contains: `*.ipynb` Jupyter Notebooks.
- Key files: `pipeline.ipynb` in each directory for testing specific OCR models.

**wikipedia-scraped/**
- Purpose: Sub-project for gathering ground truth data.
- Contains: Scraping scripts and local datasets.
- Key files: `03_wikipedia_scrape.ipynb`.

## Key File Locations

**Entry Points:**
- `election_pipeline/dags/election_dag.py` - Production pipeline entry.
- `typhoon/pipeline.ipynb` - Manual testing entry point for VLM extraction.

**Configuration:**
- `pyproject.toml` - Global project settings.
- `election_pipeline/.env` - Environment secrets.
- `election_pipeline/src/config.py` - Static configuration and master candidate lists.

**Core Logic:**
- `election_pipeline/src/processor.py` - Image transformation and API calls.
- `election_pipeline/src/ocr_parser.py` - Data extraction and validation rules.

**Testing:**
- No formal test suite. Interactive testing is conducted via `.ipynb` files in experimental directories.

## Naming Conventions

**Files:**
- `snake_case.py` - Python modules and scripts.
- `kebab-case.ipynb` - Experimental notebooks.
- `UPPERCASE.md` - GSD documentation/artifacts.

**Directories:**
- `snake_case` / `kebab-case` - Varied, but production code uses `snake_case`.

## Where to Add New Code

**New Pipeline Feature:**
- Logic: Add to `election_pipeline/src/` as a submodule.
- Orchestration: Update `election_dag.py`.

**New Experiment:**
- Create a new top-level directory (e.g., `experiment-name/`).
- Use Jupyter Notebooks for prototyping.

**New Master Data:**
- Update `election_pipeline/src/config.py`.

## Special Directories

**.planning/**
- Purpose: Internal state and documentation for the Get Shit Done (GSD) framework.
- Source: Generated by GSD mapping and planning commands.
- Committed: Yes.

**election_pipeline/output_data/**
- Purpose: Storage for generated CSV and JSON results.
- Committed: No (usually gitignored).
