### ⚙️ Pipeline Architecture

Pipeline นี้ทำหน้าที่แปลงไฟล์ PDF รายงานผลการนับคะแนน (ที่เขียนด้วยลายมือและตาราง) ให้กลายเป็นข้อมูลดิจิทัลที่สะอาดและพร้อมใช้งาน (Clean Data) โดยแบ่งการทำงานเป็น 5 ขั้นตอนหลัก:

1. **Ingestion & Optimization:**
   * ระบบจะวิ่งเข้าไปในโฟลเดอร์แต่ละตำบลและหน่วยเลือกตั้ง
   * หากเป็นเอกสาร **"แบ่งเขต"** ระบบจะตัดรูปครึ่งบน-ครึ่งล่าง (Image Chunking) และแปลงเป็นภาพขาวดำ เพื่อลดขนาดไฟล์ ป้องกันปัญหา 504 Timeout
2. **OCR Extraction:**
   * ส่งภาพที่บีบอัดแล้วไปให้โมเดล `Typhoon OCR` อ่านข้อความ
   * มีระบบ **Retry 3 ครั้ง** หากอินเทอร์เน็ตสะดุดหรือ API ค้าง ระบบจะลองใหม่เองอัตโนมัติ
3. **Parsing & Data Cleaning:**
   * ใช้ Regex กวาดหาตัวเลขจำนวนบัตรต่างๆ และคะแนนในตาราง
   * แปลงเลขไทยเป็นเลขอารบิก ลบอักขระขยะทิ้ง และแปลงค่าว่าง/ขีด (`-`) ให้เป็น `0` เสมอ เพื่อให้เป็น Integer ล้วน
4. **Validation & Auto-Correction:**
   * นำชื่อที่ OCR อ่านได้ไปเทียบกับ Master Data หากพิมพ์เพี้ยนเล็กน้อย (ความคล้าย $\ge$ 80%) ระบบจะ **"ปัดชื่อ"** ให้ถูกต้องอัตโนมัติ (Fuzzy Matching)
   * คำนวณสมการคณิตศาสตร์เพื่อรีเช็คความถูกต้องของตัวเลขที่อ่านมา (Cross-check)
5. **Exporting:**
   * บันทึกไฟล์ข้อมูลรายหน่วยลงโฟลเดอร์แยกทันที (Atomic Save) เพื่อป้องกันข้อมูลหายหากโปรแกรมดับกลางคัน
   * สร้างไฟล์ **Master Summary Log** เป็นตารางสรุปว่าหน่วยไหนมีปัญหาต้องให้คนเข้ามาดูบ้าง

### 🚩 สรุปความหมายของ Validation Logs (Flags)

ระบบ Flags ถูกออกแบบมาแบบ **One-Hot Encoding** (เป็น True/False แยกคอลัมน์) เพื่อให้ทีม Operation (Human) สามารถดึงไฟล์ (Query) ไปตรวจสอบเฉพาะจุดที่มีปัญหาได้อย่างรวดเร็ว

#### 1. `needs_manual_check` (ต้องตรวจสอบด้วยคนหรือไม่?)
* **ความหมาย:** เป็นตัวสับสวิตช์หลัก (Master Switch) หากมี Flag ย่อยตัวใดตัวหนึ่ง (ด้านล่าง) มีค่าเป็น `True` ตัวนี้จะกลายเป็น `True` ทันที
* **ต้องไปเช็คตรงไหน:** หากเป็น `False` แปลว่าข้อมูลสมบูรณ์ 100% เอาไปใช้ต่อได้เลย แต่ถ้าเป็น `True` ให้ดูที่คอลัมน์ `details` เพื่อดูว่ามันแจ้งเตือนเรื่องอะไร แล้วค่อยไปดู Flag ย่อยต่อไปนี้ครับ

#### 2. `flag_math_total_used` (สมการรวมบัตรผิดปกติ)
* **ความหมาย:** เช็คจากสมการ `[จำนวนบัตรที่ใช้] = [บัตรดี] + [บัตรเสีย] + [ไม่เลือกใคร]` หากผลรวมไม่เท่ากัน จะติด `True`
* **สาเหตุที่เกิดได้:** 1. OCR อ่านตัวเลขบนหัวกระดาษผิด (เช่น อ่านเลข 5 เป็น 6)
  2. กกต. ประจำหน่วย "บวกเลขด้วยมือผิด" ตั้งแต่ตอนเขียนลงกระดาษ
* **ต้องไปเช็คตรงไหน:** ให้เปิดไฟล์ภาพ PDF ของหน่วยนั้น ดูที่ **"ส่วนหัวของกระดาษ (ข้อ 2)"** เช็คตัวเลข บัตรที่ใช้, บัตรดี, บัตรเสีย, ไม่เลือกใคร ว่าของจริงเขียนเลขอะไรไว้ แล้วแก้ให้ตรงครับ

#### 3. `flag_math_valid_score` (สมการรวมคะแนนผิดปกติ)
* **ความหมาย:** เช็คจากสมการ `[จำนวนบัตรดี] = [ผลรวมของคะแนนทุกพรรค/ผู้สมัครในตารางรวมกัน]` หากไม่เท่ากัน จะติด `True`
* **สาเหตุที่เกิดได้:**
  1. OCR อ่านคะแนนพรรคใดพรรคหนึ่งผิด หรืออ่านข้ามบางบรรทัดไป
  2. กกต. ประจำหน่วยนับคะแนนในตารางรวมกันแล้วไม่ตรงกับช่องบัตรดีด้านบน
* **ต้องไปเช็คตรงไหน:** ให้เปิดไฟล์ภาพ PDF ดูเทียบระหว่าง **"ช่องจำนวนบัตรดี (ด้านบน)"** กับ **"ตารางคะแนน (ด้านล่าง)"** ลองกดเครื่องคิดเลขรวมคะแนนในตารางดูว่าตกหล่นที่พรรคไหนครับ

#### 4. `flag_name_mismatch` (ชื่อพรรค/ผู้สมัครอ่านเพี้ยนเกินรับได้)
* **ความหมาย:** ชื่อที่ OCR อ่านมาได้ มีความคล้ายกับรายชื่อใน Master Data **น้อยกว่า 80%** (เช่น ลายมือเขี่ยมากจน AI อ่านเป็นภาษาต่างดาว หรือพนักงานเขียนชื่อย่อแปลกๆ) ระบบจึงไม่กล้า "ปัดชื่อ" ให้อัตโนมัติ
* **ต้องไปเช็คตรงไหน:** ให้ดูที่คอลัมน์ `details` ใน Log มันจะบอกว่า "ชื่อเพี้ยนมาก(<80%): [ชื่อที่อ่านได้]" จากนั้นให้คุณเปิดไฟล์ PDF ดูใน **"ตารางคะแนน"** ว่าบรรทัดที่ AI อ่านเพี้ยนนั้น จริงๆ แล้วคือพรรคอะไร หรือผู้สมัครคนไหน แล้วทำการแก้ไข (Map Data) กลับเข้าไปด้วยตัวเองครับ
