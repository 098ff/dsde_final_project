# Insights README

เอกสารฉบับนี้อธิบายรายละเอียดของแต่ละ Insight ที่ได้จากการวิเคราะห์ข้อมูลการเลือกตั้ง ซึ่งนำข้อมูลมาจากทั้ง OCR PDF ในกระบวนการทำ Data Pipeline (`election_pipeline`) และข้อมูลที่ได้จากการ Scrape จากแหล่งภายนอก (`scraping`) 

## ภูมิหลังและแรงบันดาลใจของโปรเจกต์ (Project Background)

โปรเจกต์นี้พุ่งเป้าไปที่การวิเคราะห์ผลการเลือกตั้งในพื้นที่ **จังหวัดอุทัยธานี เขต 2** เป็นหลัก ซึ่งเป็นพื้นที่ที่ **นายชาดา ไทยเศรษฐ์** จาก **พรรคภูมิใจไทย** ครองตำแหน่งชนะการเลือกตั้งมาอย่างต่อเนื่อง (รวมถึงในอดีตหลายๆ ปีที่ผ่านมา) ด้วยบริบททางการเมืองของพื้นที่นี้และกระแสข่าวลือเชิงลบต่างๆ ที่เกี่ยวข้อง ทำให้เกิดข้อสงสัยเกี่ยวกับความโปร่งใสในกระบวนการเลือกตั้ง

คณะผู้จัดทำจึงตั้งใจนำข้อมูล (Data) มาเป็นเครื่องมือเพื่อสะท้อนความน่าสงสัยเหล่านี้ให้ชัดเจนและเป็นรูปธรรมมากยิ่งขึ้น (Data-driven Insights) โดยเริ่มตั้งแต่กระบวนการดึงข้อมูลผลการเลือกตั้งระดับหน่วย (OCR) การดึงข้อมูลบริบทของพรรคการเมือง (Scraping) จนนำไปสู่การสร้างข้อค้นพบ (Insights) ทั้ง 3 ประเด็นที่มุ่งเน้นไปที่การตรวจจับความผิดปกติที่อาจเชื่อมโยงกับการซื้อเสียง และการประเมินอิทธิพลทางการเมืองในพื้นที่เป้าหมายนี้

---

## Insight 1: การตรวจจับการซื้อเสียง (Vote Buying Detection)

**1. ข้อมูลที่ใช้:**
*   ข้อมูลผลคะแนนเลือกตั้งบัญชีรายชื่อระดับหน่วยเลือกตั้ง จาก OCR PDF (`election_pipeline`) ที่ผ่านการ Manual Check แล้ว
*   ข้อมูลพิกัด (Latitude, Longitude) ของแต่ละหน่วยเลือกตั้ง สำหรับพล็อตลง Map (นำมาจาก Google Maps Geocoding API)
*   ข้อมูลบริบทของพรรคการเมืองที่ได้จากการ Scrape (`scraping`) เช่น ข้อมูลโซเชียลมีเดีย สถิติสาขาและตัวแทนพรรค เป็นต้น

**2. ที่มาของ Insight:**
*   เกิดจากข้อสังเกตความผิดปกติที่ว่า ในบัญชีรายชื่อบางหน่วยเลือกตั้ง กลับมี "พรรคเล็ก" หรือ "พรรค No Name" ที่ไม่ได้ทำแคมเปญหาเสียงในพื้นที่ ได้คะแนนสูงขึ้นมาแบบงงๆ และเมื่อเทียบกับพฤติกรรมในอดีตมักจะมีความเชื่อมโยงกับบางสิ่ง

**3. ทำไมต้องทำ Insight นี้:**
*   เพื่อสร้างเครื่องมือตรวจสอบความโปร่งใส โดยแสดงผลบนแผนที่ให้เห็นทุกหน่วยเลือกตั้ง และชี้เป้าหมายให้เห็นว่าหน่วยใดบ้างที่มีความเสี่ยง หรือมีแนวโน้มที่จะมีการกระทำผิดกฎหมายเลือกตั้ง (การซื้อเสียง)

**4. หลักการคำนวน/สูตรทางคณิตศาสตร์:**
การวิเคราะห์แบ่งออกเป็น 2 ขั้นตอนหลัก:
*   **ขั้นตอนที่ 1: การคำนวณว่าพรรคใดเป็น "พรรคเล็ก (No Name)"** โดยไม่ใช้ความรู้สึก (Instinct) แต่ใช้ตัวแปร 8 ตัว ได้แก่:
    1.  *สัดส่วน สส. เขตที่พรรคส่ง* / จำนวนที่นั่ง สส.เขต ในสภา [0, 1]
    2.  *สัดส่วน สส. บัญชีรายชื่อที่พรรคส่ง* / จำนวนที่นั่ง สส.บัญชีรายชื่อ ในสภา [0, 1]
    3.  *จำนวนสาขาพรรค:* ทำ Min-Max Scaling เป็น [0, 1]
    4.  *จำนวนตัวแทน:* ทำ Min-Max Scaling เป็น [0, 1]
    5.  *จำนวนสมาชิกพรรค:* เนื่องจากข้อมูลเบ้ จึงใช้ Log Transform ก่อน แล้วทำ Min-Max Scaling เป็น [0, 1]
    6.  *จำนวน Followers ใน Facebook:* (ถ้าไม่มีคิดเป็น 0) ใช้ Log Transform แล้วทำ Min-Max Scaling เป็น [0, 1]
    7.  *จำนวนการถูกพูดถึงใน Google Trends ในช่วง 30 วัน:* หาผลรวม 30 วัน แล้วทำ Min-Max Scaling เป็น [0, 1]
    8.  *ประวัติศาสตร์การได้ สส.:* ถ้าพรรคนั้นเคยได้อย่างน้อย 10 เสียงในรอบที่ผ่านๆ มา ถือว่าไม่ No Name (ค่า 1.0) ที่เหลือเป็น (ค่า 0.0)
    
    > **การประมวลผล:** นำตัวแปรทั้ง 8 มาทำ **PCA (Principal Component Analysis)** ลดเหลือ 1 Dimension (ทดสอบเทียบกับ K-Means แล้วพบว่า PCA ให้ผลลัพธ์ที่ดีกว่า) จากนั้นตัด Threshold ที่ประมาณ 50% ด้วยสายตา 
    > **ผลลัพธ์:** ไฟล์ `small_party.csv` ระบุว่าพรรคนั้นเป็นพรรคเล็กหรือไม่ (column: small = 0 หรือ 1)

*   **ขั้นตอนที่ 2: การตรวจจับหน่วยที่คะแนนน่าสงสัย**
    > **การประมวลผล:** ตรวจสอบคะแนนบัญชีรายชื่อของทุกพรรคในหน่วยนั้นๆ (ข้ามหน่วยที่ไม่มีคะแนนปาร์ตี้ลิสต์แต่แรก) หากมีพรรคที่ถูกจัดว่าเป็นพรรคเล็ก ได้คะแนนเกิน **6.5%** ของคะแนนในหน่วย จะถือว่าหน่วยนั้นน่าสงสัย 
    > **ผลลัพธ์:** ไฟล์ `suspect.csv` สรุปหน่วยที่น่าสงสัย (มี 4 columns)

**5. Background ของ Insight:**
*   เวลาที่มีการซื้อเสียง ผู้ทุจริตมักจะให้คำแนะนำผู้ลงคะแนนแบบง่ายๆ คือ "ให้กาเลขเดียวกันทั้ง 2 ใบ" (บัตรเขต และ บัตรบัญชีรายชื่อ) เพื่อป้องกันความสับสน ผลที่ตามมาคือ พรรค No Name ในบัตรบัญชีรายชื่อที่ดันจับสลากได้เบอร์เดียวกับผู้สมัคร สส.เขต คนที่ซื้อเสียง จะได้รับอานิสงส์คะแนนโหวตบัญชีรายชื่อตามไปด้วยอย่างผิดสังเกต

## Insight 2: การจำแนกรูปแบบความผิดปกติของคะแนน (Vote Buying Anomaly & Ballot Confusion)

**1. ข้อมูลที่ใช้:**
*   ไฟล์ `merged_parties_with_ratio.csv` (จัดเก็บในโฟลเดอร์ `/visualization_prep_insight2/`) ประกอบด้วยคะแนนของพรรคการเมืองในระบบเขตและบัญชีรายชื่อ

**2. ที่มาของ Insight:**
*   สืบเนื่องจาก Insight 1 ที่ตรวจพบความผิดปกติ Insight นี้จึงถูกออกแบบมาเพื่อสร้างภาพนิทัศน์ (Visualization) ให้เห็นความผิดปกติที่หลุดจากการกระจายตัวตามธรรมชาติ (Outlier) ชัดเจนขึ้น

**3. ทำไมต้องทำ Insight นี้:**
*   เพื่อพล็อตความสัมพันธ์ของทั้ง 2 ระบบให้เห็นด้วยตาเปล่า และเพื่ออธิบายแยกแยะสาเหตุความผิดปกติออกจากกัน ว่าคะแนนที่แปลกไปนั้นเกิดจาก "การจงใจทุจริต" หรือเกิดจาก "ความสับสนของประชาชนเอง"

**4. หลักการคำนวน/สูตรทางคณิตศาสตร์:**
*   **คำนวณอัตราส่วน:** `Ratio = คะแนนบัญชีรายชื่อ / (คะแนน สส.เขต + 1)`
*   **ค้นหาจุดผิดปกติ (Outlier):** ใช้ **Z-Score** ในการคัดกรองพรรคที่อยู่นอกเกณฑ์สถิติ
*   **การพล็อตกราฟ:** สร้าง Scatter Plot (x, y)
    *   `x` = คะแนนเขต
    *   `y` = คะแนนบัญชีรายชื่อ
    *(หมายเหตุ: พรรคการเมืองปกติควรจะเกาะกลุ่มอยู่บนเส้นแนวทะแยง y = x)*
*   **การจำแนกประเภทความผิดปกติด้วยสี:**
    *   🔴 **จุดสีแดง (Vote Buying Anomaly):** คะแนนปาร์ตี้ลิสต์สูงผิดธรรมชาติ โดยได้เยอะกว่า สส. เขต เกินกว่า 1,000 เท่าหรืออยู่ในโซนบนซ้ายของกราฟ (เช่น พรรคเพื่อชาติไทย, พลังธรรมใหม่) สะท้อนร่องรอยกลยุทธ์ "การซื้อเสียงแบบเจาะจงเฉพาะปาร์ตี้ลิสต์" หรือการกาเลขเดียวกันสองใบ
    *   🟢 **จุดสีเขียว (Ballot Confusion / Donkey Voting):** พรรคที่ได้เบอร์ต้นๆ ในบัตร (เช่น เบอร์ 1-5) มีอัตราส่วนคะแนนปาร์ตี้ลิสต์พุ่งสูงอย่างผิดสังเกต เกิดจากประชาชนนำเบอร์ผู้สมัครเขตที่ชื่นชอบ ไปเผลอกาซ้ำในบัตรบัญชีรายชื่อโดยไม่ได้ดูชื่อพรรคให้ดี หรือกาใบแรกๆ ทิ้งไป

**5. Background ของ Insight:**
*   การแยกพฤติกรรมการลงคะแนนมีความสำคัญมาก นอกจากการซื้อเสียงแล้ว เรายังพบพฤติกรรม Human Error ที่เรียกว่า "Donkey Voting" (การเลือกเบอร์แรกๆ ในบัตรไว้ก่อนเพราะความสับสนในระบบบัตรสองใบที่เบอร์ไม่ตรงกัน)

## Insight 3: วัดความศรัทธาต่อพรรคภูมิใจไทยในแต่ละตำบล

**1. ข้อมูลที่ใช้:**
*   ไฟล์ `all_districts_bhumjaithai_ratio.csv` (จัดเก็บในโฟลเดอร์ `/visualization_prep_insight3/`) มีข้อมูลอัตราส่วนคะแนนบัญชีรายชื่อต่อตำบล
*   ข้อมูลพิกัด/Shapefile ของแต่ละตำบล เพื่อนำไปแสดงเป็นแผนที่ภูมิศาสตร์ (Choropleth Map)

**2. ที่มาของ Insight:**
*   พรรคภูมิใจไทยเป็นพรรคที่มีการจัดตั้งฐานเสียงในระดับพื้นที่ได้อย่างเหนียวแน่นและมีนัยสำคัญ Insight นี้จึงเกิดขึ้นเพื่อเจาะดูอิทธิพลเชิงพื้นที่ของพรรค

**3. ทำไมต้องทำ Insight นี้:**
*   เพื่อให้สามารถดูระดับความเข้มแข็งของฐานเสียงได้ว่า "ชนะขาด" หรือ "ชนะไม่ขาด" และวัดระดับความศรัทธาของประชาชนในพื้นที่ระดับไมโคร (ตำบล) ออกมาเป็นภาพแผนที่เฉดสีที่เข้าใจง่าย

**4. หลักการคำนวน/สูตรทางคณิตศาสตร์:**
*   คำนวณจากคะแนนบัญชีรายชื่อของพรรคภูมิใจไทยในตำบลนั้นเทียบกับคะแนนรวม
*   กำหนดระดับความศรัทธา (Shade สี) โดยอิงตามค่าทางสถิติ: **ใช้ค่า ควอไทล์ (Q1, Q2, Q3) เป็น Threshold**
    *   ถ้าสัดส่วนคะแนนสูงกว่า Q3 (กลุ่มคะแนนสูงสุด) จะถูกระบายด้วย **สีน้ำเงินเข้ม** (ชนะขาด)
    *   ถ้าสัดส่วนคะแนนต่ำลงมาใน Q2 หรือ Q1 จะถูกระบายด้วยสีน้ำเงินที่ **โทนอ่อนลงมา** ตามลำดับ

**5. Background ของ Insight:**
*   ในทางการเมืองระดับท้องถิ่น การจะดูเพียงผลคะแนน สส. เขตอาจไม่ได้สะท้อนความผูกพันของประชาชนที่มีต่อตัวพรรคได้อย่างแท้จริง โดยเฉพาะในกรณีของ **จังหวัดอุทัยธานี เขต 2** ที่อยู่ภายใต้อิทธิพลของตระกูลใหญ่ การวิเคราะห์จากคะแนนบัญชีรายชื่อในระดับที่ลึกถึงตำบล จึงช่วยตอกย้ำและพิสูจน์ให้เห็นถึงภาพ "บ้านใหญ่" หรือเขตอิทธิพลเชิงพื้นที่ (Stronghold) ของพรรคภูมิใจไทยและนายชาดา ไทยเศรษฐ์ ได้อย่างชัดเจนและมีน้ำหนักมากที่สุดด้วยข้อมูลเชิงประจักษ์
